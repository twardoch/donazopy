# 05. Credential and Configuration Model

## TLDR

Credentials are loaded from environment variables or ignored local configuration and are never committed, printed, or logged.

## Sources

The default credential source is environment variables. Later releases may support ignored local config files under user config directories. Project-local `.env`, `.pypirc`, provider token files, and private research files must remain untracked.

## Validation

Each provider adapter declares required credential names. The CLI can report whether credentials appear present without printing values. Write commands must validate credentials before preparing provider calls.

## Redaction

Logs, exceptions, dry-run output, and debug traces must redact authorization headers, token query parameters, API keys, account secrets, and provider-specific secret fields. Redaction tests must cover common failure surfaces.

## Acceptance Criteria

- Provider metadata lists credential names.
- Missing credentials fail before network calls.
- Secret values are never exposed in normal output.
