# 07. Read, Export, and Dump Workflows

## TLDR

Read workflows collect provider state and write stable zone files for backup, audit, and migration.

## Provider Reads

DNS-hosting providers should support listing zones and records when their APIs allow it. Registrar providers should support reading delegation and domain status where available.

## Zone Dumps

Provider record state should be converted into normalized zone files with `$ORIGIN`, `$TTL`, SOA strategy notes where the provider omits SOA, and stable record ordering. Dumps should avoid inventing data that was not provided.

## Native Export

Some providers expose native zone-file export endpoints. Donazopy should prefer native export when it is complete and documented, then validate the exported zone through the same zone engine.

## Acceptance Criteria

- Exported zones validate locally when provider data is complete.
- Missing SOA or NS information is reported clearly.
- Provider read failures are typed and actionable.
