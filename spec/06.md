# 06. CLI Experience

## TLDR

The CLI uses Fire for discoverable commands that map directly to common DNS workflows.

## Command Groups

Initial commands include version reporting, provider listing, provider details, and zone validation. Planned commands include zone dump, zone diff, provider export, provider apply, migration planning, nameserver reassignment, and delegation verification.

## UX Principles

Commands should show exact files, providers, zones, and capabilities involved. Dry-run and plan outputs should be human-readable and suitable for saving in CI logs. Write commands must require explicit confirmation unless a trusted automation flag is used.

## Errors

Errors should explain the operation, provider, zone, and next action. Provider API errors should preserve status codes and request IDs where safe while redacting secrets.

## Acceptance Criteria

- `donazopy --help` works after installation.
- `donazopy providers` lists provider keys.
- `donazopy provider cloudflare` returns useful metadata.
- `donazopy validate-zone <path>` validates a local zone file.
