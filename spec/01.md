# 01. Vision and Scope

## TLDR

Donazopy is a Python CLI for managing DNS zones, DNS-provider records, and registrar nameserver delegation with a zone-file-first workflow.

## Goals

Donazopy should let operators parse and edit BIND-style zone files, export provider DNS state into zone files, apply zone files to provider DNS APIs, and reassign authoritative nameservers through registrar APIs. It should work as an audit, migration, backup, and synchronization tool rather than a hidden magic layer over DNS.

## Non-Goals for the First Release

The first release should not attempt a universal provider implementation with full live-write coverage for every provider. It should scaffold the provider registry, implement the local zone engine, provide safety-first workflows, and add provider adapters only where official API behavior is verified by documentation and tests.

## Scope Boundary

DNS hosting and registrar operations are distinct. Updating NS records inside a zone file changes child-zone data. Updating delegation at the parent registry requires a registrar/domain API. Donazopy must model these operations separately.

## Acceptance Criteria

- The CLI exposes project and provider discovery commands.
- Zone files can be validated locally before provider writes.
- Provider modules are isolated and advertise capabilities.
- Destructive or delegation-changing workflows are plan-first and verifiable.
