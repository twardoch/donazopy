# 11. Testing Strategy

## TLDR

Testing starts with deterministic local zone tests, then mocked provider contracts, then opt-in live integration tests.

## Unit Tests

Unit tests cover zone parsing, serialization, normalization, diffing, provider registry metadata, CLI command dispatch, credential validation, and error formatting.

## Mocked Provider Tests

Each provider adapter should have mocked HTTP tests for authentication headers, request bodies, pagination, API validation errors, rate limits, and idempotent no-change behavior.

## Live Tests

Live tests must be opt-in through explicit environment variables and disposable test zones. They must never run by default in CI. They should create, verify, update, verify, delete, and verify cleanup where providers permit it.

## Acceptance Criteria

- Local tests run without network access.
- Provider write behavior is not enabled without mocked tests.
- Live tests are gated and documented.
