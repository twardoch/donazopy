# 08. Write, Import, and Sync Workflows

## TLDR

Write workflows transform a validated zone file into an explicit provider change plan before applying API calls.

## Planning

The apply workflow reads local zone state, reads provider state, computes a plan, and reports creates, updates, deletes, no-ops, warnings, and blocking errors. Deletions and delegation-affecting changes are unsafe by default.

## Applying

When a plan is accepted, the provider adapter performs API calls in a safe order. For record CRUD providers, creates and updates should happen before deletes when possible. For native zone import providers, export-before-import backups are required.

## Idempotency

Applying the same normalized zone repeatedly should converge without unnecessary changes. Provider-specific normalization must prevent churn caused by response ordering, implicit TTLs, or value formatting differences.

## Acceptance Criteria

- Every write-capable command supports dry-run.
- Plans can be reviewed before changes are applied.
- Unsafe changes are blocked unless explicitly confirmed.
- Provider writes are followed by verification reads when APIs support them.
