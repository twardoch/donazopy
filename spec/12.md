# 12. Implementation Roadmap

## TLDR

Build Donazopy in safety-first slices: scaffold, zone engine, provider contracts, read flows, write plans, provider adapters, and delegation workflows.

## Phase 1: Scaffold and Specification

Create Hatch/uv packaging, Fire CLI entry point, provider registry placeholders, local zone validation, tests, build scripts, full specification, TODO backlog, and changelog. This phase establishes architecture without pretending provider writes are implemented.

## Phase 2: Zone Engine

Implement normalized records, stable serialization, diffing, fixture coverage, and CLI commands for local zone inspection and normalization.

## Phase 3: Provider Protocols

Define DNS-hosting and registrar interfaces, credentials, errors, dry-run plans, and mocked adapter test harnesses.

## Phase 4: High-Confidence Providers

Implement Cloudflare, IONOS, Joker, AWS Route 53, DNSimple, and one DNS-only provider first because research indicates useful API coverage.

## Phase 5: Migration and Delegation

Add export/apply/migrate/delegation workflows with verification and rollback guidance.

## Phase 6: Provider Expansion

Add remaining providers behind capability gates, using official docs and mocked tests before enabling writes.

## Acceptance Criteria

- Each phase leaves a working CLI and passing tests.
- Provider write support is never marked complete before docs and tests confirm behavior.
- Completed TODO items are moved into `CHANGELOG.md` after verification.
